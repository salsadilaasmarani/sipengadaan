Dokumentasi SiPengadaan

SiPengadaan adalah sistem informasi pengadaan barang (purchasing) yang dibangun di atas framework PHP custom dengan pola arsitektur MVC. Project ini dikembangkan sebagai tugas akhir semester (UAS) mata kuliah Pengembangan Aplikasi Web Berbasis Framework.

ℹ️
Versi Framework: 1.0.0 (Development)  ·  PHP: 8.2+  ·  MySQL: 8.0+
Pengantar

Instalasi

Ikuti langkah-langkah berikut untuk menjalankan SiPengadaan di lingkungan lokal:

1. Clone Repository

bash
git clone https://github.com/username/sipengadaan.git
cd sipengadaan

2. Konfigurasi Database

Salin file konfigurasi dan sesuaikan dengan environment lokal:

bash
cp config/database.example.php config/database.php
config/database.php
<?php
return [
    'host'     => 'localhost',
    'port'     => '3306',
    'database' => 'sipengadaan_db',
    'username' => 'root',
    'password' => '',
    'charset'  => 'utf8mb4',
];

3. Jalankan Migration

bash
php sipengadaan migrate
php sipengadaan seed    # (opsional) isi data dummy

4. Jalankan Server

bash
php sipengadaan serve
# Server berjalan di http://localhost:8000

Routing

Semua route didefinisikan di config/routes.php. Framework mendukung method GET, POST, PUT, dan DELETE.

config/routes.php
<?php
use Core\Router;

// Auth
Router::get('/login',  'AuthController@showLogin');
Router::post('/login', 'AuthController@login');
Router::get('/logout', 'AuthController@logout');

// Purchase Request
Router::get('/pr',           'PurchaseRequestController@index');
Router::get('/pr/create',    'PurchaseRequestController@create');
Router::post('/pr',          'PurchaseRequestController@store');
Router::get('/pr/{id}',      'PurchaseRequestController@show');
Router::post('/pr/{id}/submit', 'PurchaseRequestController@submit');

// Approval
Router::get('/approval',        'ApprovalController@index');
Router::post('/approval/{id}',  'ApprovalController@decide');

// Vendor
Router::get('/vendor',       'VendorController@index');
Router::get('/vendor/create','VendorController@create');
Router::post('/vendor',      'VendorController@store');
Modul

Daftar Modul

01

Purchase Request (PR)

Modul pertama untuk mengajukan permintaan pengadaan barang. User dengan role requester dapat membuat PR baru, mengisi detail barang yang dibutuhkan, dan mengirimkan untuk disetujui.

GET /pr — Daftar semua PR
POST /pr — Buat PR baru
GET /pr/{id} — Detail PR
POST /pr/{id}/submit — Kirim untuk approval
02

Approval Workflow

Alur persetujuan bertingkat. Setiap PR yang disubmit akan masuk ke antrian approval sesuai level jabatan. Approver dapat menyetujui atau menolak dengan catatan.

GET /approval — Antrian approval
POST /approval/{id} — Setuju / Tolak
03

Vendor Management

Pengelolaan data supplier/vendor. Menyimpan informasi kontak, kategori barang, rating, dan riwayat transaksi untuk perbandingan penawaran.

GET /vendor — Daftar vendor
POST /vendor — Tambah vendor
GET /vendor/{id} — Detail vendor
04

Purchase Order (PO)

Generate PO secara otomatis setelah PR disetujui. Tracking status pengiriman barang dan konfirmasi penerimaan oleh tim gudang.

GET /po — Daftar PO
POST /po — Buat PO dari PR
POST /po/{id}/receive — Konfirmasi terima
05

Laporan & Analitik

Dashboard ringkasan pengeluaran, grafik tren pengadaan per bulan, dan ekspor laporan ke format PDF atau Excel untuk kebutuhan pelaporan manajemen.

GET /report/dashboard — Dashboard
GET /report/export?format=pdf — Export PDF
Database

Skema Database

Berikut adalah tabel-tabel utama dalam sistem SiPengadaan beserta relasinya:

SQL — tabel utama
-- Tabel Users
CREATE TABLE users (
  id         INT PRIMARY KEY AUTO_INCREMENT,
  name       VARCHAR(100) NOT NULL,
  email      VARCHAR(100) UNIQUE NOT NULL,
  password   VARCHAR(255) NOT NULL,
  role       ENUM('admin','requester','approver','purchasing') DEFAULT 'requester',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Tabel Purchase Request
CREATE TABLE purchase_requests (
  id          INT PRIMARY KEY AUTO_INCREMENT,
  pr_number   VARCHAR(20) UNIQUE NOT NULL,
  requester_id INT REFERENCES users(id),
  title       VARCHAR(200) NOT NULL,
  description TEXT,
  total_amount DECIMAL(15,2),
  status      ENUM('draft','pending','approved','rejected','po_created') DEFAULT 'draft',
  created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Tabel PR Items
CREATE TABLE pr_items (
  id         INT PRIMARY KEY AUTO_INCREMENT,
  pr_id      INT REFERENCES purchase_requests(id),
  item_name  VARCHAR(200) NOT NULL,
  quantity   INT NOT NULL,
  unit       VARCHAR(50),
  est_price  DECIMAL(15,2)
);

-- Tabel Vendor
CREATE TABLE vendors (
  id         INT PRIMARY KEY AUTO_INCREMENT,
  name       VARCHAR(200) NOT NULL,
  contact    VARCHAR(100),
  email      VARCHAR(100),
  phone      VARCHAR(20),
  address    TEXT,
  rating     DECIMAL(3,2) DEFAULT 0,
  is_active  TINYINT(1) DEFAULT 1
);

-- Tabel Purchase Order
CREATE TABLE purchase_orders (
  id         INT PRIMARY KEY AUTO_INCREMENT,
  po_number  VARCHAR(20) UNIQUE NOT NULL,
  pr_id      INT REFERENCES purchase_requests(id),
  vendor_id  INT REFERENCES vendors(id),
  total      DECIMAL(15,2),
  status     ENUM('created','sent','received','cancelled') DEFAULT 'created',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

Menjalankan Migration

Framework menyediakan CLI tool untuk mengelola migrasi database:

bash
# Jalankan semua migrasi
php sipengadaan migrate

# Rollback migrasi terakhir
php sipengadaan migrate:rollback

# Reset semua tabel dan jalankan ulang
php sipengadaan migrate:fresh

# Isi data dummy
php sipengadaan seed
